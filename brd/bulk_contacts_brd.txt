Document Purpose
Capture the complete, testable requirements for Club Contacts Bulk Upload. Includes GDPR decisions, verification/consent behaviour, list semantics, and suppression rules. Ready for build once Open Questions are resolved.
Project Summary
Ingest offline contact CSVs, create accounts silently for new contacts, send verification to new contacts only, enforce verification before club outreach, attach to Lists, and capture Marketing/Lotto consent obtention. Uploaded contacts may be added to Active/
Expired
 lotto lists when lotto consent is attested.
Business Goals &amp; Objectives
The primary objective is to enable club admins and PROs to reach their entire community, including offline contacts who haven’t purchased or created accounts online, through a compliant, reliable bulk-import workflow. We will maintain strict data hygiene by preventing duplicate account creation; require verification before any email can be received to improve deliverability and compliance; and map an admin’s consent attestations to each contact’s (Account holder’s) MyAccount preferences (Marketing, Lotto notifications). In addition, when lotto consent is provided at upload, uploaded contacts may be added to the Active 
or Expired
 lotto players lists and are eligible to receive lotto results accordingly. Together, these objectives ensure clubs can communicate with both online and offline audiences while preserving trust, compliance, and data quality.

Functional Requirements
FR-001 - Access Control &amp; Entry to Bulk Import (Must)
The system must restrict access to the Bulk Import area to authorised users only and expose it via a single “Bulk Import” entry point. SU + BO feature flags.
FR-002 - Bulk Import Launcher: Choose Import Type (Must)
The system must present a launcher screen with three options: Contacts, Membership registrations, Lotto playslips, and route accordingly.
FR-003 - CSV Upload &amp; Validation (Must)
The system must accept a fixed‑schema CSV (A=First Name, B=Last Name, C=Email; header required), validate all rows, display invalid rows inline with reasons, and block progression until the dataset is 100% valid.
FR-004 - Silent Account Creation &amp; Verification for New Contacts (Must)
The system must silently create accounts for emails not found in the system and send a verification email to those new contacts only; existing contacts are not notified after upload.
FR-005 - Lists: Static &amp; Lotto/ 50/50 Lists (Must)
The system must allow creating a Static List 
and/or adding uploaded contacts to existing Lists, including Active/Expired lotto players when Lotto consent is attested at upload.
FR06
 - Consent Attestation (batch) &amp; MyAccount Mapping
Admin can provide consent for General club news and club marketing and/or Lotto / Draw notifications for uploaded club contacts. Store audit (uploader id, timestamp, consent types). Apply flags after verification. Users can change preferences in MyAccount.
FR-
007
 - MyAccount Preferences &amp; Migration Defaults (Must)
The system must expose per-club preferences (Club comms master, Marketing, Lotto notifications) and define secure defaults for existing account holders.
FR-008 - Send‑time Eligibility (Must)
The system must evaluate recipients against verification, consent, purpose, deliverability, and lotto rules with reconciling totals (Matched = Eligible + Suppressed).
FR-009
 - Compose Emails: Default Marketing, Eligibility Counts, and Transactional Override (Should)
The system must default once-off and recurring club emails to General club news and marketing, compute and display recipient eligibility (matched total vs eligible total) based on consent/verification rules, and allow a Transactional override that sends to all matched recipients only after explicit confirmation by the sender.
FR-010 - Split existing Lotto and Draw players system list
The system must replace the existing single System List: “Lotto players” with two distinct System Lists: “Lotto players - active” and “Lotto players - expired”, and ensure these lists are selectable and behave correctly when sending a message. The same for Draw (50/50) players.
FR-011 - System-Defined Dynamic Recipient List: “All club contacts” (Must)
The system must provide a read-only, system-defined dynamic recipient list called “All club contacts”. This list automatically aggregates every unique contact associated with the club from any source, including: paid membership purchasers, Registered members, lotto ticket purchasers, 50/50 draw purchasers, shop purchasers, Club users (club owners, club admins, group managers), plus second parents/guardians added via the Connect app. The list is continuously updated and de-duplicated during send. The list is visible in the recipient dropdown for both Once-off and recurring emails and cannot be manually edited
FR-012 — Admin Confirmation Email on Successful Bulk Upload (Must)
The system must send a confirmation email to the uploading admin after a successful Contacts bulk upload, summarising the import, consents, list actions, and next steps, without exposing contact PII in the email body.
